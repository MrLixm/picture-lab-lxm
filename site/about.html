about

This website provides resources to evaluate and compare the different picture formation algorithm available online. Those kind of algorithm are also referred as:

  • display rendering transform
  • view transform
  • LUT

In few words, picture formation is converting physical quantities to a readable visual representation. The concept can be explored further with:

  • https://github.com/sobotka/scise/wiki/Picture-Formation
  • https://hg2dc.com/2025/03/13/question-34/
  • https://github.com/jedypod/open-display-transform/wiki/doc-introduction

The code source for this website and the image generation can be retrieved at https://github.com/MrLixm/picture-lab-lxm

Assets

Is referred by 'asset' the single image file of a specific scene.

Most of the assets were retrieved from the web where they were kindly made accessible by their author. All rights belong to their respective authors, and they are only shared here for education purposes.

The original image file are not shared on the source GitHub repository yet, but it's possible to download them yourself by checking their side-car metadata .json file.

Renderers

A renderer correspond to a color-transform that must be applied on pixel data. All renderers are color-managed for an sRGB output but will showcase a different picture formation algorithm.

ℹ️ Note that the preferred sRGB transfer-function is 2.2 when possible, but some Renderer may use the piecewise function.

The color transforms are serialized in an OpenColorIO configuration, which will be ingested by OpenImageIO's oiiotool for transforming the pixel data.
